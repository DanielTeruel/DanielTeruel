## Featured Project

### [AD Migration Lab — On-Premises Active Directory to Azure](https://github.com/DanielTeruel/ad-migration-azure)

![Banner](https://raw.githubusercontent.com/DanielTeruel/ad-migration-azure/main/banner.png)

End-to-end migration of a Windows Server AD environment to Microsoft Azure, following the **Microsoft Cloud Adoption Framework (CAF)**. Full infrastructure built on-premises, then migrated workload by workload to Azure with IaC, CI/CD, and professional documentation throughout.

| Phase | Status |
|---|---|
| On-premises infrastructure — DC01 · APP01 · WS001 | ✅ Complete |
| Hybrid onboarding — Azure Arc · Update Manager · Bastion · Intune | ✅ Complete |
| Azure identity — Entra Connect · RBAC · MFA · Hybrid Join | ✅ Complete |
| Azure networking — VNet · NSG · subnets | ✅ Complete |
| Azure SQL VM — vm-sql01 · DanielDB migrated | ✅ Complete |
| Azure App Service — live · Key Vault · Managed Identity · VNet Integration | ✅ Complete |
| Azure Files — DC01 SharedFiles migrated via AzCopy | ✅ Complete |
| Azure Backup — Recovery Services Vault · SQL workload backup | ✅ Complete |
| Security & monitoring — Azure Policy · Log Analytics · App Insights · alerts | ✅ Complete |
| IaC — ARM Template · Bicep · Terraform | ✅ Complete |
| CI/CD — GitHub Actions · automated deploy on git push | ✅ Complete |

**On-Premises:** AD DS · DNS · DHCP · WSUS · GPO · IIS · ASP.NET Core 8 · SQL Server Express · Windows Server Backup

**Azure:** Entra ID · Entra Connect · Azure Arc · Azure Update Manager · Intune · App Service · Azure VM + SQL Server 2022 · Key Vault · Managed Identity · Azure Files · Recovery Services Vault · Azure Bastion · Azure Policy · Log Analytics · Application Insights

**IaC & Automation:** Bicep · Terraform · ARM Templates · GitHub Actions · Azure CLI · AzCopy

### [daniel-portfolio-web](https://github.com/DanielTeruel/daniel-portfolio-web)

ASP.NET Core 8 portfolio web application — the workload migrated in the AD Migration Lab. Hosted on **Azure App Service (B1)**, connected to SQL Server on an Azure VM via VNet Integration and Key Vault Managed Identity. Deployed automatically via **GitHub Actions** on every push.
